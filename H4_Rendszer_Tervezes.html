Bevezetés

Ez a dokumentum alapvetően azt a célt szolgálja, hogy részletesen meghatározza a rendszer felépítését, komponenseit és azok kapcsolatait, figyelembe véve a projekt specifikus követelményeit.

Célok és alapelvek

A projekt központi célja, hogy egyedi, műfajilag sokszínű játékot hozzon létre, amely ötvözi a sci-fi, western és fantasy elemeket retro pixel art stílusban. A rendszertervezés során ezeket a célokat úgy kell figyelembe venni, hogy biztosítani tudjuk az optimális működést és a felhasználói elvárások teljesítését.

A rendszertervezés szerepe

  • Meghatározza a játék működési logikáját.
  • Bemutatja a komponensek kapcsolódási pontjait.
  • Últalános irányelveket ad az implementációhoz.

A rendszertervezési folyamat során nem alkalmazunk adatbázisokat, mivel a projekt kizárólag a játékállások mentéséhez használ perzisztens adatokat. Ezzel a megközelítéssel egyszerűsítjük a rendszertervezést, miközben biztosítjuk a teljesítényt és rugalmasságot.

2. Rendszer architektúrája

2.1 Architektúra Áttekintés

A rendszer moduláris felépítésű, és az alábbi fő komponensekre tagolódik:

  • Game Engine: A játék alapvető működtetéséért felelős komponens, amely kezeli a játék mechanikáit, fizikai interakciókat és a szintek közötti váltásokat.
  • Felhasználói Interfész: A játékos és a rendszer közötti kommunikáció biztosítása, beleértve a menük, HUD (Heads-Up Display), és értesítések kezelését.
  • Karakterrendszer: A játékosok, NPC-k és ellenségek adatainak és interakcióinak kezeléséért felelős alrendszer.
  • Térképek és Szintek: A játék környezeteit definiáló térképek és szintek létrehozása és kezelése.

2.2 Komponensek Közötti Kommunikáció

Az egyes komponensek közötti adatátadás eseményalapú és interfészorientált megközelítést követ:

  • Game Engine ↔ Karakterrendszer: Az eseménykezelők segítségével a Game Engine értesíti a karaktereket az állapotváltozásokról (pl. mozgás, támadás).
  • Game Engine ↔ Térképek és Szintek: A Game Engine felelős a térképek betöltéséért, frissítéséért és az objektumok pozicionálásáért a szinteken.
  • Karakterrendszer ↔ Felhasználói Interfész: A karakterek állapotai (pl. egészség, sebzés) frissítik a HUD kijelzőt.

4. Tervezés

4.1 Osztálydiagramok és Komponensek

4.1.1 Game Engine

Attribútumok:

  • currentLevel: Level
  • isPaused: bool

Metódusok:

  • loadLevel(levelID: int): void
  • update(deltaTime: float): void
  • handleEvent(event: Event): void
  • saveGameState(): void
Példa kód - Szint betöltése
    public void LoadLevel(int levelID)
    {
        currentLevel = LevelManager.GetLevel(levelID);
        currentLevel.Load();
        isPaused = false;
    }

4.1.2 Karakterrendszer

Attribútumok:

  • player: Player
  • enemies: List<Enemy>
  • npcList: List<NPC>

Metódusok:

  • spawnCharacter(type: CharacterType, position: Vector2): Character
  • removeCharacter(character: Character): void
  • updateCharacterStates(): void
  • processInteractions(): void
Példa kód - Karakter létrehozása
    public Character SpawnCharacter(CharacterType type, Vector2 position)
    {
        Character newCharacter = CharacterFactory.Create(type);
        newCharacter.Position = position;
        if (type == CharacterType.Enemy)
        {
            enemies.Add((Enemy)newCharacter);
        }
        else if (type == CharacterType.NPC)
        {
            npcList.Add((NPC)newCharacter);
        }
        return newCharacter;
    }

4.1.3 Térképek és Szintek

Attribútumok:

  • mapData: Map
  • spawnPoints: List<Vector2>
  • activeObjects: List<GameObject>

Metódusok:

  • loadMap(mapID: int): void
  • resetMap(): void
  • updateMapState(deltaTime: float): void
  • checkLevelCompletion(): bool
Példa kód - Térkép frissítése
    public void UpdateMapState(float deltaTime)
    {
        foreach (var obj in activeObjects)
        {
            obj.Update(deltaTime);
        }
        CheckLevelCompletion();
    }
    
    private bool CheckLevelCompletion()
    {
        return activeObjects.All(obj => !obj.IsObjective || obj.IsCompleted);
    }

4.2 Algoritmusok és Adatszerkezetek

4.2.1 Játékállás Mentése

A játékállás mentése JSON formátumban történik, amely tartalmazza a játékos állapotát, a szintinformációkat és az aktív objektumokat.

Példa kód - Játékállás mentése
    public void SaveGame(string filePath)
    {
        var saveData = new SaveData
        {
            PlayerState = player.GetState(),
            LevelState = currentLevel.GetState(),
            MapState = mapData.GetState()
        };
        File.WriteAllText(filePath, JsonConvert.SerializeObject(saveData));
    }

4.2.2 Karakter Interakciók Kezelése

Az interakciók eseményvezérelt rendszerrel működnek. Például a játékos interakcióba lép egy NPC-vel, vagy ellenséggel kerül szembe.

Példa kód - Interakció kezelése
    public void InteractWithNPC(NPC npc)
    {
        if (npc.IsQuestGiver)
        {
            npc.OfferQuest(player);
        }
        else if (npc.HasTradeOption)
        {
            npc.OpenTradeMenu(player);
        }
    }

4.2.3 Térképek Frissítése

A térképek dinamikusan frissülnek, például objektumok mozgatásával vagy új objektumok hozzáadásával.

Példa kód - Térkép frissítése
    public void UpdateMap(float deltaTime)
    {
        foreach (var obj in activeObjects)
        {
            obj.UpdateState(deltaTime);
        }
        SpawnNewObjects();
        CheckCollisions();
    }

4.2.4 Küldetések Frissítése

A küldetések állapota folyamatosan frissül, és értesíti a játékost a teljesített feladatokról.

Példa kód - Küldetések frissítése
    public void UpdateQuests()
    {
        foreach (var quest in activeQuests)
        {
            if (quest.IsCompleted)
            {
                player.GrantReward(quest.Reward);
                quest.MarkAsFinished();
            }
        }
    }

4.2.5 Fegyverkezelés

A fegyverkezelési rendszer biztosítja a fegyverek használatát, elhasználódását, állapotuk követését és javítását.

Példa kód - Fegyver támadás

    {
        if (weapon.IsReady && target != null)
        {
            target.TakeDamage(weapon.Damage);
            weapon.DecreaseDurability();
        }
    }
Példa kód - Fegyver javítása
    public void RepairWeapon(Weapon weapon, int repairAmount)
    {
        weapon.Durability += repairAmount;
        weapon.IsReady = true;
    }
Példa kód - Fegyver tulajdonságok lekérdezése
    public string GetWeaponStats(Weapon weapon)
    {
        return $"Name: {weapon.Name}, Damage: {weapon.Damage}, Durability: {weapon.Durability}/{weapon.MaxDurability}";
    }

4.2.6 Erőforrás-kezelés

Az erőforrások (pl. lőszer, mana, energia) kezelését a rendszer nyomon követi, hogy biztosítsa a játékosok megfelelő korlátozását és a játék egyensúlyát.

Példa kód - Erőforrás használata
    public bool UseResource(string resourceType, int amount)
    {
        if (resources[resourceType] >= amount)
        {
            resources[resourceType] -= amount;
            return true;
        }
        return false;
    }

Egy adott erőforrás kimerülése esetén a rendszer figyelmezteti a játékost, hogy töltsék újra vagy keressenek alternatív megoldásokat.

Példa kód - Erőforrás újratöltése
    public void ReplenishResource(string resourceType, int amount)
    {
        resources[resourceType] += amount;
    }

4.2.6 Életerő kezelés

Az életerő rendszer a játékosok és ellenségek egészségét kezeli, beleértve a sebzések és gyógyítások dinamikáját.

Példa kód - Sebzés alkalmazása
    public void TakeDamage(Character character, int damage)
    {
        character.Health -= damage;
        if (character.Health <= 0)
        {
            character.IsAlive = false;
        }
    }
Példa kód - Gyógyítás
    public void Heal(Character character, int healAmount)
    {
        character.Health += healAmount;
        if (character.Health > character.MaxHealth)
        {
            character.Health = character.MaxHealth;
        }
    }
Példa kód - Életerő állapot kijelzése
    public string DisplayHealthStatus(Character character)
    {
        return $"{character.Name}: {character.Health}/{character.MaxHealth} HP";
    }

4.2.7 Mozgások

A karakterek mozgásának kezelése a játéktéren belül, beleértve az akadályokat és különleges mozgási képességeket.

Példa kód - Mozgás végrehajtása
    public void MoveCharacter(Character character, Vector2 direction, float speed)
    {
        Vector2 newPosition = character.Position + direction * speed;
        if (IsValidPosition(newPosition))
        {
            character.Position = newPosition;

    }
Példa kód - Különleges mozgások

        Vector2 dashPosition = character.Position + direction * distance;

            character.Position = dashPosition;
        }
    }

4.2.9 Interakciós mechanizmusok

A játékos és a környezet közötti interakciókat az eseményvezérelt rendszer kezeli. Példák lehetnek ajtók kinyitása, tárgyak felvétele vagy kapcsolók aktiválása.

Példa kód - Interakció indítása
    public void Interact(EnvironmentObject obj)
    {
        if (obj is Door door)
        {
            door.Open();
        }
        else if (obj is Item item)
        {
            player.Inventory.Add(item);
            item.RemoveFromWorld();
        }
    }
Példa kód - Interakció vizsgálata
    public bool CanInteract(EnvironmentObject obj)
    {
        return obj.IsInteractable && Vector2.Distance(player.Position, obj.Position) < interactionRange;
    }

4.2.11 Ellenséges AI

Az ellenségek mesterséges intelligenciája különböző viselkedéseket kezel, például követést, támadást, vagy menekülést.

Példa kód - Játékos követése
    public void ChasePlayer(Enemy enemy, Player player, float speed)
    {
        Vector2 direction = (player.Position - enemy.Position).Normalized();
        enemy.Position += direction * speed * Time.deltaTime;
    }
Példa kód - Támadási logika
    public void AttackPlayer(Enemy enemy, Player player)
    {
        if (Vector2.Distance(enemy.Position, player.Position) < attackRange)
        {
            player.TakeDamage(enemy.Damage);
        }
    }

4.2.13 Részletes eseménykezelő rendszer

Az eseménykezelő rendszer lehetővé teszi az események regisztrálását, kezelését és kiváltását különböző komponensek között.

Példa kód - Esemény regisztrálása
    public void RegisterEvent(string eventName, Action callback)
    {
        if (!events.ContainsKey(eventName))
        {
            events[eventName] = new List();
        }
        events[eventName].Add(callback);
    }
Példa kód - Esemény kiváltása
    public void TriggerEvent(string eventName)
    {
        if (events.ContainsKey(eventName))
        {
            foreach (var callback in events[eventName])
            {
                callback.Invoke();
            }
        }
    }

5. Nem Funkcionális Követelmények

5.1 Teljesítmény

  • Játéksebesség: A játék a célplatformokon (PC, konzolok) legalább 60 FPS teljesítményt nyújtson, stabil működéssel, még magas részletességi beállítások mellett is.
  • Betöltési idők: A szintek közötti betöltési idő ne haladja meg az 5 másodpercet.
  • Skálázhatóság: A rendszer képes legyen kezelni különböző hardveres teljesítményszinteket, alacsony grafikai beállítások támogatásával a régebbi rendszerek számára.

5.2 Kompatibilitás

  • Platformok: A játék támogatja a Windows, macOS és Linux operációs rendszereket.
  • Hardverigény: Az alkalmazás futtatásához legalább 4 GB RAM, integrált vagy dedikált grafikus processzor szükséges.
  • Input támogatás: Az alkalmazás kompatibilis legyen a billentyűzet+egér, valamint különböző gamepad típusokkal (Xbox, PlayStation).

5.3 Biztonság

  • Mentési rendszer: A mentett állások védelme érdekében a JSON fájlokat digitális aláírással kell ellátni, hogy megakadályozzuk a manipulációt.
  • Adatvédelem: A játék nem gyűjt személyes adatokat, és minden statisztikai információ anonimizált.

5.4 Használhatóság

  • Felhasználói élmény: Az interfészeknek intuitívnak kell lenniük, a célközönség igényeihez igazodva.
  • Hozzáférhetőség: Opcionálisan biztosítani kell nagyobb betűméreteket és színkontraszt-beállításokat a vizuális nehézségekkel küzdők számára.
  • Testreszabhatóság: A játékosok képesek legyenek a vezérlési beállításokat és grafikai opciókat egyénileg testre szabni.

5.5 Stabilitás

  • Hibakezelés: A rendszer automatikusan jelentse a hibákat, naplózza azokat, és ha szükséges, értesítse a fejlesztőket vagy a felhasználót.

5.6 Hibajavítási iterációk

A hibajavítási iterációk biztosítják a rendszer folyamatos fejlesztését és stabil működését. Az alábbi lépések írják le a hibajavítási folyamatot:

  1. Hiba bejelentése: A hibákat a rendszer automatikusan naplózza, és tárolja helyi log fájlokban. Felhasználók manuálisan is bejelenthetik a hibákat egy külön hibajelentő felületen keresztül.
  2. Hiba azonosítása: A naplófájlok elemzése során a fejlesztők azonosítják a hiba forrását. A hibák súlyossági szintek szerint kategorizálódnak: kritikus, közepes, alacsony prioritású.
  3. Elsődleges javítás: Kritikus hibák esetén azonnali javítás történik a rendszer stabilitásának megőrzése érdekében. Nem kritikus hibáknál javítási feladat kerül a fejlesztési iterációk sorába.
  4. Tesztelés: Az elsődleges javítás után a hibát kiváltó helyzetekre vonatkozó teszteket futtatnak. Új teszteseteket hoznak létre a hiba megismétlődésének megelőzése érdekében.
  5. Verziófrissítés: A hibajavításokat a következő frissítés részeként adják ki, amely tartalmazza a dokumentált módosításokat és a kapcsolódó tesztelési eredményeket.
  6. Visszacsatolás: A felhasználók visszajelzést küldhetnek a javítás hatékonyságáról. Ezeket az információkat a fejlesztési folyamat optimalizálására használják.

5.7 Stabilitás és redundancia

A rendszer stabilitása érdekében redundáns mechanizmusokat kell bevezetni, amelyek biztosítják, hogy kritikus hibák esetén is működőképes maradjon.

  • Biztonsági mentések: Automatikus biztonsági mentések készítése minden jelentős változtatás előtt.
  • Folyamatfigyelés: Olyan mechanizmusok, amelyek figyelik a rendszer állapotát, és előre jelezhetik a lehetséges hibákat.
  • Crash Recovery: Az összeomlás utáni automatikus helyreállítás, amely minimalizálja a felhasználói adatok elvesztését.

5.8 Skálázhatóság

A rendszernek képesnek kell lennie a különböző hardveres teljesítményszinteken való működésre, a felhasználók igényeitől függően.

  • Hardveroptimalizáció: Különböző grafikai és teljesítménybeállítások támogatása, hogy a rendszer futtatható legyen gyengébb hardvereken is.
  • Hálózati skálázhatóság: Többjátékos funkció esetén a rendszer támogatja a felhasználószám dinamikus növekedését.
  • Felhasználói igények kezelése: A rendszer képes legyen növekvő terhelés (pl. nagyobb mentési fájlok) hatékony kezelésére.

5.9 Rendszerfrissítések és karbantartás

A rendszer rendszeres frissítéseket kap, amelyek javításokat, új funkciókat és optimalizációkat tartalmaznak. A karbantartási folyamat biztosítja a hosszú távú működőképességet.

  • Frissítések ütemezése: A frissítések meghatározott időközönként kerülnek kiadásra, minimalizálva a felhasználói élmény zavarását.
  • Automatikus frissítések: A rendszer automatikusan letölti és telepíti a kisebb frissítéseket a felhasználó engedélyével.
  • Kompatibilitási ellenőrzések: A frissítések telepítése előtt a rendszer ellenőrzi a jelenlegi konfigurációval való kompatibilitást.

5.10 Kompatibilitás és platformfüggetlenség

A rendszernek több platformon is egységesen kell működnie, figyelembe véve az operációs rendszerek és hardveres konfigurációk közötti eltéréseket.

  • Platformok támogatása: A játék támogatja a Windows, macOS és Linux operációs rendszereket, valamint a konzolokra történő optimalizációt.
  • Input eszközök: A rendszer támogatja a billentyűzetet, egeret, valamint különféle gamepadokat (Xbox, PlayStation).
  • Grafikai beállítások: Az alacsony, közepes és magas szintű grafikai beállítások biztosítják a rugalmasságot különböző hardveres környezetekben.

5.11 Adatvédelem és biztonság
